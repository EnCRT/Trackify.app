# Документация: Bluetooth (BLE) интеграция Trackify и Moto Lap Timer

Данная документация описывает функционал передачи трек-файлов между аппаратным GPS логером (ESP32) и мобильным приложением (Flutter) посредством Bluetooth Low Energy (BLE).

---

## 1. Функционал GPS логера (ESP32 - Trackify.gnss_module)

### Переход в режим BLE
- По умолчанию логер при наличии спутников находится в режиме ожидания (READY) или ведет запись (LOGGING).
- **Активация режима:** Долгое нажатие (около 4 секунд) на единственную физическую кнопку устройства активирует **Скрытый режим Bluetooth**.
- **Индикация:** 
  - На OLED дисплее появляется надпись "BLE Server", ожидание подключения.
  - Светодиод (NeoPixel) загорается назначенным цветом (например, мерцающий синий или фиолетовый) для индикации готовности.
- **Особенности скрытого режима:**
  - В отличие от WiFi, который вещает открытую сеть, BLE может быть настроен без постоянного Advertisement (радиовещания) или с вещанием ограниченной мощности.
  - Устройство рекламирует себя под именем `Trackify`.
- **Выход из режима:** Повторное короткое нажатие на кнопку возвращает логер в стандартный режим работы (IDLE/READY), BLE сервер отключается.

### Протокол работы
Логер реализует два BLE Characteristic в рамках кастомного Service UUID:
1. **Command Characteristic:** принимает текстовые команды от телефона:
   - `LIST` — запрос списка файлов на SD-карте.
   - `GET /filename.txt` — запрос на скачивание конкретного файла.
2. **Data Characteristic:** используется для возврата данных на телефон (посредством Notifications):
   - Список файлов передается в виде строк.
   - Данные файла передаются фрагментами (чанками), размер которых зависит от согласованного MTU (до 512 байт).

## 2. Функционал Мобильного Приложения (Flutter - moto_lap_timer)

### Интерфейс пользователя (UI)
1. **Кнопка BT (Bluetooth):**
   - Находится в нижней навигационной панели (Bottom Navigation Bar) рядом с кнопкой добавления трека (`+`).
   - Иконка — стандартный значок Bluetooth.
2. **Окно поиска и синхронизации:**
   - При нажатии на кнопку BT открывается модальное окно или новый экран.
   - Приложение запрашивает права на использование Bluetooth (и геолокации, что обязательно для сканирования BLE на мобильных ОС).
   - Выполняется автоматический поиск устройства с именем `Trackify`.
3. **Список файлов:**
   - После успешного подключения приложение отправляет команду `LIST` и отображает полученные файлы.
   - Каждый элемент списка содержит:
     - Имя файла (например, `log_001.txt`).
     - Размер файла в мегабайтах (МБ).
     - Кнопку **"Скачать"**.
4. **Процесс загрузки:**
   - При нажатии "Скачать" иконка заменяется на `CircularProgressIndicator` (круговой индикатор).
   - Внутри индикатора или рядом отображается процент загрузки, рассчитанный на основе полученных байт и общего размера файла.
5. **Интеграция с созданием заезда:**
   - Как только загрузка завершается на 100%, файл сохраняется в локальное временное хранилище телефона.
   - Приложение автоматически вызывает `GpsParserService`, парсит скачанный файл и открывает экран `RaceCreationScreen`.
   - Пользователь видит скачанный трек на карте и может приступить к разметке финишной линии или анализу.

### Ограничения
- **Скорость:** BLE оптимизирован для передачи небольших объемов данных с низким энергопотреблением. Для файлов размером в несколько мегабайт загрузка может занять от нескольких десятков секунд до пары минут.
- **Удержание экрана:** Во время передачи файла рекомендуется не сворачивать приложение, чтобы операционная система не оборвала BLE соединение.

## Взаимодействие (Сценарий использования)

1. Мотоциклист завершает заезд.
2. Подходит к трекеру (Trackify) и зажимает кнопку на 4 секунды. Трекер переходит в режим BLE (загорается фиолетовый индикатор).
3. На телефоне мотоциклист открывает приложение **Moto Lap Timer** и нажимает иконку **Bluetooth** в нижней панели.
4. Приложение мгновенно находит трекер, подключается и выводит список заездов.
5. Мотоциклист видит последний файл (например, `log_012.txt`, размер `1.2 МБ`) и нажимает "Скачать".
6. Через ~40 секунд файл скачан, приложение автоматически открывает экран анализа трека (`RaceCreationScreen`).
7. Пользователь размечает круг и сохраняет заезд. Трекер можно выключить или перевести в обычный режим коротким нажатием.
